# v0.2 — The Journey Era

## Thesis

Praxis v0.2 transforms a curriculum into a world. v0.1 built measurement. v0.2 builds place.

---

## Taxonomy

| Term | Meaning | Scope |
|---|---|---|
| Challenge | Any individual learning activity | Universal |
| Stage | Pedagogical domain | Movement, Search, Editing, etc. |
| Family | Related activities within a stage | Stage-scoped, explicitly ordered |
| Layer | Learning purpose | Tutorial, Training, Trial, Boss |
| Region | UX identity of a stage | Frontend-only, presentation |
| Journey | User progression through curriculum | Computed by Journey Engine |
| Hub | Primary entrypoint | Neovim UI |

A challenge belongs to exactly one stage, exactly one family, and exactly one layer.

---

## Architecture Decisions

- **CLI owns truth. Neovim owns experience.** Go computes. Lua presents. Never inverted.
- **No standalone TUI.** The editor is the arena.
- **`plans/` documents intent.** Not UI mockups, not visual concepts, not narratives.
- **Existing tutorials are sacred.** No removal, rewrites, validator changes, or replay regressions across v0.2.x.

---

## Journey Model

| Step | Layer | Experience |
|---|---|---|
| Arrival | Onboarding | Welcome, orientation, first action, first success |
| Practice | Tutorial | Single-concept drills, highly guided |
| Synthesis | Training | Combined skills, family-based, composite evaluation |
| Proof | Trial | Multi-skill pressure, constrained execution |
| Application | Boss | Realistic tasks, meaningful application |

Onboarding is a frontend-only flow, not a layer. It is not curriculum, not persistent, not measured.

---

## Invariants

- 41/41 replay tests must pass throughout v0.2.x.
- The Hub always answers *what should I do next?*
- No infrastructure ships before its consumer.
- No feature may require accounts, servers, cloud state, or external services.
- Attempts, Completions, Success Rate, Mastery, and Confidence remain the authoritative measurement model.

### v0.2.1 — Arrival (shipped)

**Question:** Where do I begin?

**Answer:** Welcome. Start here.

**Delivered:**
- Lua frontend extracted into focused modules (challenge.lua, session.lua, ui.lua, onboarding.lua, hub.lua)
- First-time onboarding flow: `:Praxis` → welcome → Enter → `motion_rush`
- Returning users experience v0.2.0 behavior unchanged
- 0 Go changes, 0 schema changes, 0 validator changes

**Deferred to v0.2.2:** Hub surface, Continue Journey, `praxis next`, region/stage language, progress displays, result screen navigation

### v0.2.2 — Journey (shipped)

**Question:** Where am I?

**Answer:** Every challenge belongs to a stage of learning.

**Delivered (Phase 1):**
- `internal/content/curriculum.go` — production metadata source of truth (Metadata struct, 41-entry map, MetadataFor lookup, ValidStages)
- Taxonomy tests rewritten to validate against MetadataFor (coverage, valid stages, unique concept/context pairs, stage order, layer consistency)
- `praxis next` — returns next un-Practiced challenge ID in curriculum order
- `praxis stage <id>` — returns stage name for a given challenge ID

**Deferred to Phase 2:** Hub surface, Continue Journey, result screen navigation, progress displays

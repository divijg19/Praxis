# Roadmap

This document describes high-level directional intent. It is not a commitment. Priorities may shift based on what the repository needs at each stage.

## v0.0.x — Foundation Era (complete)

**41 challenges, 27 tests, 2 validators.**

Curriculum breadth. Architecture hardening. Testing maturity.

- v0.0.1–v0.0.15: Content accumulation
- v0.0.16: Validation architecture
- v0.0.17–v0.0.18: Editing curriculum
- v0.0.19: Registers + testing maturity
- **v0.0.20: Foundation Freeze** — documentation, API contracts, replay automation

The v0.0.x era is deliberately closed at v0.0.20.

## v0.1.x — Precision Mode Era

**Theme:** Quality over quantity.

No new curriculum packs. Instead: precision, mastery, and feedback.

- Challenge quality: better content, clearer instructions, tighter validation
- Precision mode: stricter success criteria, timing, accuracy tracking
- Mastery tiers: bronze through grandmaster progression
- UI/UX improvements for the challenge experience
- Core engine stability for the next phase

## v0.2.x — Neovim Integration Era

**Theme:** Shape the plugin, validate the workflow.

- Neovim plugin improvements: commands, mappings, feedback, statusline
- Plugin validation: installation, upgrade, compatibility testing
- Frontend quality: error messages, edge cases, session management
- Documentation: plugin usage, configuration, troubleshooting

## v0.3.x — TUI / Dashboard Era

**Theme:** Make Praxis visible.

- Standalone TUI: browse challenges, track progress, see curriculum
- Dashboard: statistics, mastery progress, session history
- Cross-session persistence
- The interface layer that makes Praxis feel like a complete product

## Beyond

The specific shape of v1.0 will depend on what v0.3.x reveals about user needs. The engine architecture is designed to support any frontend (TUI, Neovim, web, API) without changing challenge content.
